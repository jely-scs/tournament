Программа для проведения турниров. 


Требования: 

1) Программа должна без боли запускаться на разных компьютерах (в расчет берем ОС Windows) 

2) Программа должна работать в офлайне с возможностью синхронизироваться с онлайн ресурсами

3) Нужно взаимодействие с сайтом Мастер Клинка

4) дополнительно можно заинтегрироваться с программкой-табло (обсудить, нужно ли)


На текущий момент выбраны технологии:

Рисуем странички на php, они должни загружаться с интернет-площадки и разворачиваться на локальной машине. 

Используемая база данных - xml

Взаимодействие с сайтом через загрузку xml-файлов.


Первая страница: 

создание турнира - дата, название, тип (парное-одиночное оружие), лига (классика, премьер, ТЧ) 

Выбор параметров турнира! Параметрами могут быть способ подсчета очков, тип отборочных (один раунд- несколько), тип финала

Параметры собираем в шаблоны - отдельно, не перед турниром


Вторая страница:

Загрузка списка участников (с сайта выгружается общий список, потом отмечаются участники), нужен поиск участников + возможность добавления новых участников. 

Каждый участник - фио, клуб, лига

Список участников сгруппирован по лиге

Распределение участников на группы (если нужно)

Картинка страницы!


Страница групповые бои:

слева - список участнков с текущими баллами

Справа - окно формирования пар. 

Пары формируются по рейтингу, равный с равным. При этом учитываются следующие факторы: 

1) разница в очках не более n 

2) из разных клубов

2) в этом турнире не еще не бились


Отдельный вариант - сформировать бои каждый с каждым для каждой группы.


Многораундовые бои: 3-5 раундов, в итоговый рейтинг идет разница по раундам + очки за победу. Между раундами учитывается разница очков. 

Уточнить еще раз про раунды (будут ли в ближайших турнирах раунды?)!


При нечетном количестве бойцов следующая схема: в первом круге боец не бьется, во втором круге бьется дважды, в третьем круге на выбор - не бьется совсем или выбирается противник вручную.


После 3-х кругов нужна возможность выбрать из списка бойцов тех, кто проходит в финал. (в этот момент повляются бойцы с одинаковым рейтингом, они бьются за выход, но в программке можно не учитывать эти бои, просто отмечать тех, кто прошел)


Отдельная страница: финал.

Правила формирования пар для финала. Может быть количество человек (в порядке вероятности): 8, 12, 4, 6.

Бои идут на вылет. Правила формирования пар:

1) Сильный со слабым

2) разница в очках n=0 

3) разные клубы 

(пояснение про пункты 2 и 3, одноклубников разводим только при наличии кандидата из другого клуба с ровно такими же очками)

если остается 4 бойца - последний круг: сильный со слабым по рейтингу из отборочных, но бойцы не выбывают, а формируются следующие пары: бой за первое-второе места, бой за третье-четвертое места. 

если остается 3 бойца - пары составляются каждый с каждым.


Записываем победителей турнира - первые 3 места.


Печать документов. По нажатию на определенную кнопку на странице формируется документ, который отправляем на печать. 

Печать для групповых боев: список пар для текущего круга.  (нужен пример шаблона) (будет использоваться секретарем и судьей)

Печать итогов групповых боев - рейтинг бойцов, возможно, нужны результаты боев по кругам. (для ознакомления бойцам)

Печать финала: список пар (для судей и секретаря). 

в формах обязательно предусмотреть поле для предупреждений: отдельно критические (красные) и замечания (желные).


Система рейтингов.

При загрузке на сайт masterklinka после турнира. в xml по каждому турниру: 

1) список победителей (1,2,3 места),

2) список участвовавших бойцов + рейтинговые баллы. Баллы считаются разными способами (модифицируемое место), текущий способ такой:

 - рейтинг из отборочных - всем.

Бонусы:

 - 3 очка за выход в финал, но не попал в 4-ку

 - 6 очков за 4-е место

 - 9 очков за 3-е место

 - 12 очков за 2-е место

 - 15 очков за 1-е место
